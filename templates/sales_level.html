Sales Levels 1–5 — What changes?

The difference between each Sales Levels along with their conversation samples.

At a glance

Levels build on each other.

Level What it adds on top Best for
Level 1 Core flows (reservation/menu/info/manage), allergy safety, reschedule-before-cancel, PDF rules. Reliable baseline support
Level 2 Reward messaging in greeting + reminders after booking/reschedule/closing. Loyalty / retention
Level 3 Priority menu upsell (premium picks) when menu is requested. Upsell without personalization
Level 4 Personalized recs before adding items to an existing reservation + priority dishes (if enabled). Higher conversion on add-ons
Level 5 First-conversation promotions question + accept natural dates exactly + priority/personalized logic combined (if enabled). Most “salesy” & dynamic

How to use this page

  • Use the tabs to switch between levels.
  • Each tab shows: “Rules that matter” + a mini conversation sample.
  • Samples intentionally highlight what changes between levels.

Level 1: Baseline support

Clean, helpful service with structured reservation and menu flows.

Greeting
Friendly, no rewards, no promos question.
Reservation workflow
Name → contact → date/time/party → confirm → availability → occasion → promo code → allergies/preferences → book.
Menu PDF rule
Send only if available/successful; otherwise guide category browsing (don’t mention PDF).
Safety
Don’t invent allergen/nutrition/protein; use tools. Offer reschedule before cancel.
What Level 1 sounds like:
Warm, short chunks, line breaks before questions.

Level 2: Rewards added

Same as Level 1, but the greeting and post-booking messaging highlight rewards (when available).

Mandatory reward in greeting
“GREAT NEWS — you have a {reward_label} on {reward_type}.”
Reward reminders
After booking confirmation + reschedule confirmations + closings.
Everything else
Reservation flow, menu PDF rule, allergy safety are unchanged from Level 1.
Key difference:
Same behavior, just reward messaging sprinkled in at specific points.

Level 3: Priority menu upsell

When someone asks for the menu, show premium/priority picks first (then offer full categories).

Menu request → priority picks first
“Here are some premium picks our guests love:” + names + prices in {currency}.
PDF behavior unchanged
Send menu PDF only if available/successful; otherwise don’t mention it.
No repeat calls
Don’t re-show priority items unless they ask for “premium/top picks” again.
Key difference:
Level 3 is where upselling becomes the default when users ask for “menu”.

Level 4: Personalized add-on recommendations

When a reservation exists and the guest wants to add items, you show personalized recommendations first.

Adding items → personalized recs first
“Based on your tastes…” + names + prices in {currency}.
Priority menu still exists (if enabled)
Menu requests can still show premium picks first, depending on settings.
Allergy check after adding
After any dish is added, ask about allergies (and verify ingredients as needed).
Key difference:
Level 4 targets upsell at the moment of commitment: “add to my reservation”.

I already booked for tonight. Can you add some dishes?

Which one would you like to add?

Allergy check triggered after adding

Level 5: Promotions question + natural dates

First conversation asks about promotions. Also accepts natural date phrases exactly (e.g., “next Friday”).

First conversation ONLY
Immediately after greeting: “We have some available promotions, do you wanna see them?”
Natural date phrases
Accept “today/tomorrow/next Friday” and pass exactly to the backend.
Priority + personalization logic combined
Menu request → priority picks. Add-to-reservation → personalized recommendations (if enabled).
Key difference:
Level 5 adds a proactive promotions hook at the start, then continues with the normal flows.
